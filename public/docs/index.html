Getting Started

Specmark helps you highlight requirements in Markdown documents and turn them into structured feedback for LLM coding agents.

1. Paste Markdown Add your spec or PRD in the editor.
2. Annotate Switch to Review mode and select text to leave specific feedback.
3. Copy Feedback Export clean Markdown for your LLM agent using the "Copy comments" button.

Share Codes

Share codes are 6-character codes that let others view your markdown document. They expire after 7 days.

To load a shared document, enter the code in the header and click "Load".

CLI Setup

Prefer the terminal? Use the CLI to create share links from the command line.

Installation

Download the script and make it executable:

curl -fsSL https://specmark.dev/cli/specmark -o specmark
chmod +x specmark
sudo mv specmark /usr/local/bin/

Requirements

  • curl - for making HTTP requests
  • jq - for parsing JSON responses

Usage

specmark <file.md>

This will upload your markdown file and return a share URL and code.

Example

$ specmark spec.md

Share created!

URL:     https://specmark.dev/?c=A2B3C4
Code:    A2B3C4
Expires: 2025-01-12 14:30

Environment Variables

SPECMARK_API_URL - Override the API endpoint (default: https://specmark.dev)

Tips

  • Annotations are saved locally in your browser (localStorage), tied to the document content.
  • Click on highlighted text in Review mode to edit or delete annotations.
  • Overlapping annotations are shown with a striped pattern.
  • The feedback export includes the highlighted text as a quote followed by your comment.

Share codes expire after 7 days. Annotations are stored locally and not shared with others.
